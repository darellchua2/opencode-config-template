# OpenCode Configuration

This file contains custom instructions for opencode to customize behavior for this environment.

## Task Delegation Workflow

For **all non-trivial tasks**, follow this 5-step workflow:

```
┌─────────────────────────────────────────────────────────────────┐
│  1. PRIMARY PLANS                                                │
│     Analyze request → Create execution plan                      │
│                             │                                    │
│                             ▼                                    │
│  2. BROKER ──────────────────────────────────────────────────┐  │
│     Invoke skill-broker-subagent                              │  │
│     Returns: { subagents: [...], skills: [...], reasoning }   │  │
│                             │                                    │
│              ┌──────────────┴──────────────┐                   │
│              ▼                              ▼                   │
│     ┌────────────────┐            ┌─────────────────┐          │
│     │ 0 subagents    │            │ ≥1 subagents    │          │
│     │ (skip to 4)    │            │ (proceed to 3)  │          │
│     └────────┬───────┘            └────────┬────────┘          │
│              │                              │                    │
│              │                              ▼                    │
│              │                   3. EXECUTE SUBAGENTS           │
│              │                   ├─ If >1 → PARALLEL execution  │
│              │                   ├─ Collect results/context     │
│              │                   └─────────────────┬────────────┘
│              │                                     │             │
│              └──────────────────────┬──────────────┘             │
│                                     ▼                            │
│              4. PRIMARY EVALUATES + USES SKILLS                  │
│                 ├─ Review subagent results                       │
│                 ├─ If incomplete → Load skills via Skill tool    │
│                 └─ Execute skill workflows directly              │
│                             │                                    │
│                             ▼                                    │
│                   5. RE-PLAN                                     │
│                 ├─ Create updated plan with new context          │
│                 ├─ If complete → respond to user                 │
│                 └─ If incomplete → loop back to step 2           │
└─────────────────────────────────────────────────────────────────┘
```

### Step 1: Primary Plans

Before invoking the broker:
1. Analyze the user's request
2. Identify the task type and scope
3. Create an initial execution plan
4. Prepare a one-sentence summary for the broker

### Step 2: Broker Invocation

```
Task tool:
  subagent_type: skill-broker-subagent
  prompt: <one-sentence task summary>
```

The broker returns:
```json
{
  "subagents": [
    { "name": "subagent1", "confidence": 0.95, "reason": "..." },
    { "name": "subagent2", "confidence": 0.92, "reason": "..." }
  ],
  "skills": [
    { "name": "skill1", "confidence": 0.88, "reason": "..." }
  ],
  "reasoning": "brief explanation"
}
```

**Both arrays can be empty `[]`** — proceed to step 4 for direct handling or skill usage.

### Step 3: Execute Subagents

**If broker returns ≥1 subagents:**

| Count | Execution Strategy |
|-------|-------------------|
| 1 subagent | Invoke via Task tool, wait for result |
| >1 subagents | Spawn ALL in parallel, collect all results |

**Parallel Execution Example:**
```
# For: "Review code, run tests, create PR"
# Broker returns 3 subagents → spawn all at once:

Task tool: code-review-subagent
Task tool: testing-subagent  
Task tool: pr-workflow-subagent

# Collect all results before proceeding to step 4
```

### Step 4: Evaluate + Use Skills

After subagent execution (or if `subagents: []`):

1. **Review results** — Check what subagents accomplished
2. **Evaluate completeness** — Is the task done?
3. **If incomplete** → Check `skills` array from broker
4. **Load relevant skills** via Skill tool:
   ```
   Skill tool:
     name: <skill_name>
   ```
5. **Execute skill workflows** directly (you handle the skill steps)

### Step 5: Re-Plan Loop

1. **Integrate new context** from subagents and skills
2. **Update execution plan** based on results
3. **Decision point:**
   - **Complete** → Respond to user with final result
   - **Incomplete** → Loop back to Step 2 (re-invoke broker with updated context)

### Trivial Tasks (skip workflow)

- Single file read
- One-liner explanation
- Simple yes/no question
- User explicitly requests direct handling

### Confidence Thresholds

When evaluating broker results:
- `confidence >= 0.7` → Proceed with handler
- `confidence < 0.7` → Ask user ONE clarifying question

## External File Loading

When you encounter a file reference (e.g., @rules/general.md), use your Read tool to load it on a need-to-know basis.

- Do NOT preemptively load all references
- When loaded, treat content as mandatory instructions
- Follow references recursively when needed

## Rationale

This workflow enables:
- **Parallel execution** of multiple subagents for efficiency
- **Clear separation** between discovery (broker) and execution (primary)
- **Iterative refinement** through the re-plan loop
- **Optimal resource use** by running independent tasks concurrently
