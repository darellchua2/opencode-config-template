# Subagent Prioritization

When a task matches a subagent's specialization, **ALWAYS delegate** rather than handling directly.

## Subagent Domains

| Subagent | Domain |
|----------|--------|
| `linting-subagent` | linting, formatting, code quality |
| `testing-subagent` | tests, coverage, specs |
| `git-workflow-subagent` | branches, commits, PRs, issues |
| `documentation-subagent` | docs, docstrings, README |
| `opentofu-explorer-subagent` | Terraform, Kubernetes, AWS, cloud |
| `nextjs-setup-subagent` | Next.js project setup |
| `coverage-subagent` | coverage badges, reports |
| `opencode-tooling-subagent` | skills, agents creation/audit |
| `refactoring-subagent` | DRY, deduplication |
| `tdd-subagent` | test-driven development |
| `image-analyzer` | screenshots, UI analysis |
| `diagram-subagent` | diagrams, flowcharts |
| `docx-creation-subagent` | Word documents, .docx files, reports, letters |
| `error-resolver-subagent` | explicit error diagnosis only |

## Decision Flow

1. Match task to subagent domain → delegate
2. No match → check skill match
3. No skill → handle directly

## Tool Restrictions

- Subagents cannot use `bash`, `task`, `question`, `todowrite`
- Read-only unless task requires writes (tests, docs, lint fixes)
- MCP access per subagent's domain only
