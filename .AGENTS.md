# Subagent Prioritization

**ALWAYS discover and evaluate subagent suitability BEFORE considering skills or handling directly.**

## Priority Order

1. **Subagents** - Check `agents/subagents/` directory first
2. **Skills** - Only if no suitable subagent found
3. **Direct** - Handle directly only as last resort

## Subagent Discovery Process

1. **Scan** `agents/subagents/` for available subagents
2. **Match** task keywords against subagent descriptions/domains
3. **Delegate** to the most suitable subagent if match found
4. **Fallback** to skills only when no subagent matches

## Tool Restrictions

- Subagents cannot use `bash`, `task`, `question`, `todowrite`
- Read-only unless task requires writes (tests, docs, lint fixes)
- MCP access per subagent's domain only
