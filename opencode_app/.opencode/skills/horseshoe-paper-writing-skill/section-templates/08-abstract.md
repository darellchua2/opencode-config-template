# Abstract Template

> **Horseshoe role:** The whole paper in ~250 words. Write it LAST, after the
> horseshoe arms are aligned.

## Four-Sentence Scaffold (structured abstract, multi-venue)

```
{PROBLEM}: {1 sentence — why this matters. Cite an engineering / industry
stake: cost / safety / downtime.}

{METHOD}: To address this, we propose {METHOD NAME}, a {ONE-LINE
DESCRIPTION}. {1–2 sentences — the key idea, sensor, model architecture.
NO EQUATIONS.} {DATASET sentence: "We construct {DATASET NAME}, comprising
{COUNT} {ITEMS} across {N CLASSES}."}

{RESULT}: Experimental results show that {BEST CONFIGURATION} achieves
{HEADLINE NUMBER} on {DATASET / TASK}, with {SECONDARY METRIC} reaching
{VALUE}. {1 sentence on generalization: out-of-sample / cross-site.}

{IMPLICATION}: These findings highlight {GENERALIZATION CLAIM, HEDGED}. The
proposed methodology demonstrates strong potential for application in
{ADJACENT DOMAIN}, contributing to {BROADER GOAL}.
```

## Word Budget

| Section | Target words |
|---------|--------------|
| Problem | 30–40 |
| Method | 80–100 |
| Result | 60–80 |
| Implication | 30–40 |
| **Total** | **200–260** |

ASCE typical: 250 words max. Elsevier / IEEE / MDPI vary (some allow 400).
Check the target venue's author guide.

## Self-Check
- [ ] No equations, no citations, no figure references.
- [ ] Headline number present (the single most important result).
- [ ] Dataset name + sample count mentioned.
- [ ] Generalization sentence (out-of-sample / cross-site) included.
- [ ] Hedged implication (no overstatement).
- [ ] Every numbered contribution from Intro ¶ 4 is **implicit** in the
      abstract — if a contribution is not represented, either the abstract
      is incomplete OR the contribution is not load-bearing (delete it).

## Mirror Reminder
The abstract is a **compressed horseshoe**: problem (Intro ¶ 1) → method
(bridge overview) → result (Conclusion ¶ 1) → implication (Conclusion ¶ 5).
If the abstract drifts from this 4-part structure, the horseshoe is broken.
