# Practical Applications / Highlights / Implications Template

> **Horseshoe role:** Practitioner-facing translation. Section name varies by
> venue (see SKILL.md §2.2 for the rename map).

| Venue family | Section name | Required? |
|--------------|--------------|-----------|
| ASCE | **Practical Applications** | Mandatory |
| Elsevier (Autom. Constr., Build. Environ., etc.) | **Highlights** (3–5 bullets) or **Implications** paragraph | Common |
| IEEE (Sensors, Access) | (omit; fold into Conclusion) | Optional |
| MDPI (Sensors, Buildings) | **Practical Implications** | Optional |
| Springer (ISARC proc.) | Optional | Optional |

## Voice & Style

- **Plain English** — no equations, no jargon, no acronyms without expansion.
- **One paragraph** (~150 words) for the prose version; 3–5 bullets for the
  Highlights version.
- Address the practitioner directly: "construction teams", "inspectors",
  "facility managers", "operators" — not "researchers".
- Include the **headline number** in plain terms.

## Scaffold (prose version — Practical Applications / Implications)

```
This study introduces a {SIMPLE DESCRIPTION OF TOOL / METHOD} to help
{PRACTITIONER ROLE} detect / predict / monitor {PROBLEM} {EARLY / CHEAPLY /
AUTOMATICALLY}. Currently, {PRACTITIONERS} often rely on {CURRENT PRACTICE},
which {LIMITATION IN PLAIN TERMS}. We developed a tool that {WHAT IT DOES,
PLAINLY} using {SENSOR / DATA SOURCE}. Even when the tool has only
{FEW-SHOT / LIMITED DATA}, it still performs well. It can recognize {RANGE
OF PROBLEMS}, even in {CHALLENGING CONDITION: noisy environments, real
sites}. This method offers a {COST / LABOR / SAFETY BENEFIT} alternative to
{CURRENT PRACTICE}. Beyond {PRIMARY DOMAIN}, this approach could benefit
{ADJACENT INDUSTRY} that faces similar challenges. By making {OUTCOME} more
accurate and less dependent on {HUMAN JUDGMENT / EXPERTISE}, this research
supports {BROADER GOAL: safer, smarter, more efficient operations}.
```

## Scaffold (Highlights version — 3–5 bullets, Elsevier)

```
- {PROBLEM STATEMENT IN PLAIN LANGUAGE}.
- {METHOD SUMMARY IN ONE LINE — NO JARGON}.
- {HEADLINE RESULT WITH A NUMBER}.
- {PRACTITIONER BENEFIT}.
- {ADJACENT DOMAIN APPLICABILITY}.
```

## Self-Check
- [ ] No equations or technical jargon.
- [ ] Practitioner role named explicitly (construction teams, inspectors,
      facility managers, operators).
- [ ] Headline benefit stated plainly (cost / labor / safety).
- [ ] Adjacent industry mentioned for broader applicability.
- [ ] Reads at a non-technical-manager level (Flesch reading ease ≥ 50).
- [ ] Section name matches target venue (§2.2 rename map).
