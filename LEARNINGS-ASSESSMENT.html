Anti-Pattern Registry

All anti-patterns from learnings that should be encoded as "do not do this" checks, grouped by target skill.

clean-code-skill   7 additions

broad-except-masks-bugs
Catch-all except Exception in auth/transport paths conflates bugs with outages. Narrow catches, let bugs propagate. 3 projects (workflow-engine, hollow-detector, frontend).
silent-failure-sequential-async
Critical async ops catching own failures with console.error prevent callers from taking corrective action. Throw or return result type.
two-phase-dataclass-initialization
Function returns dataclass with placeholder values that caller must separately patch. Silent wrong results if second call is forgotten.
method-name-reuse-different-semantics
Method named identically to existing method in another class but checks a different condition — misleading.
hardcoded-sed-version-swap
SDK version hardcoded in 3+ build files. DRY violation causing version drift.
self-documented-duplication
Code comment admits duplication ("could be replaced by X") but follow-up never lands. Convert TODO to ticket immediately.
doc-claimed-purity-vs-reality
README claims a layer is "pure" but code systematically violates it. Docs and code drift.

python-backend-skill   4 additions

module-singleton-global-mutation
global _x singleton hides coupling, hard to reset in tests, no lifecycle. Prefer FastAPI Depends().
inline-imports-in-functions
Function-body imports hide dependencies from static analysis and mask circular-import problems.
json-default-str-numpy-masking
json.dump(default=str) silently converts numpy types to strings, breaking JSON type contracts.
two-phase-dataclass-initialization
(Also listed under clean-code) Python-specific manifestation: placeholder zero values + caller-must-patch.

security-audit-skill   5 additions

auth-early-return-null-account-id
Skipping ownership check when account_id is None allows any header-less user to access any resource — privilege escalation.
encryption-key-length-not-validated
base64-valid ≠ crypto-valid. Validate decoded key length at config-load, not at first encrypt() call.
missing-tenant-isolation-definitions
Multi-tenant table without tenant_id column leaks metadata across tenants.
hardcoded-magic-timeout
HTTP client timeout hardcoded as magic number, ignoring per-node/per-request config. Not tunable without redeploy.
local-terraform-state-production
Local backend for production IaC state — no locking, no backup, no collaboration safety.

database-migration-skill   2 additions

detached-orm-across-sessions
Fetch in one SQLAlchemy session, mutate after close, commit in new session — changes silently lost.
pydantic-on-jsonb-columns
Setting Pydantic BaseModel instances on SQLAlchemy JSONB columns causes TypeError at flush time.

Good Practice Registry

All patterns/solutions from learnings that should be encoded as "do this" guidance, grouped by target skill.

security-audit-skill   2 additions

claim-check-ephemeral-secret-cache
Store decrypted secrets in TTL in-memory cache keyed by opaque UUID; pass only claim ID through workflow history; pop() (single-read) at exec time. Prevents plaintext in orchestrator history.
fail-closed-open-config-toggle
Single boolean governs fail branch: prod denies (fail-closed), dev allows (fail-open). Reinforced by circuit breaker + positive cache.

design-patterns-skill   2 additions

atomic-sql-update-race-free-transition
UPDATE ... WHERE expected_state RETURNING cols as optimistic lock — one round-trip, no extra lock table, no TOCTOU.
double-checked-locking-async-refresh
Fast-path check → acquire lock → re-check → refresh. Caps concurrent token refreshes to exactly ONE HTTP POST (prevents thundering herd).

python-backend-skill   2 additions

centralized-single-source-config
ALL env vars read in one pydantic-settings Settings class. No os.getenv() elsewhere. Validators + model_validator derive values.
defensive-enum-mapping-from-db-strings
Wrap enum lookup in try/catch returning null, then filter nulls from stream. Graceful degradation for stale/forward-incompatible DB values.

api-design-skill   2 additions

sse-backpressure-drop-connection
On queue overflow for LLM chunks, close the SSE connection (not drop oldest). LLM chunks are cumulative — losing any chunk makes output incoherent.
placeholder-swap-validation
Swap file-reference values for placeholder before JSON-Schema validation, restore originals in finally block. Validates shape without coupling to transport scheme.

authentication-authorization-skill   2 additions

stateless-cookie-cache-maxage-match-session
In stateless auth, set cookieCache.maxAge equal to session.expiresIn. If JWE expires before session, findSession() returns null and deletes the session.
cookie-only-proxy-server-rbac
Proxy should only check cookie existence (fast, local). Move token refresh to customSession, RBAC to per-route server layouts. Avoids network calls in middleware.

clean-code-skill   1 addition

isinstance-over-field-name-lists
Use isinstance(value, BaseModel) instead of hardcoded frozenset of field names. Future-proof, eliminates dead code risk.

database-migration-skill   1 addition

alembic-jsonb-requires-dicts
Alembic bulk_insert with asyncpg requires Python dicts for JSONB columns, not JSON strings.

accessibility-a11y-skill   1 addition

aria-alert-dynamic-errors
Add role="alert" to conditionally rendered error banners so screen readers announce them. Add aria-label on icon-only dismiss buttons.

performance-optimization-skill   1 addition

zero-copy-buffer-to-uint8array
For large files in Lambda: new Uint8Array(buffer.buffer, buffer.byteOffset, buffer.byteLength) creates a view, not a copy. Saves ~500MB at 500MB file size.

test-generator-framework-skill   1 addition

mock-headers-magicmock-truthy-trap
MagicMock auto-creates .headers as truthy MagicMock. .headers.get("Location") returns truthy MagicMock → false positive. Always set resp.headers = headers or {} (real dict).
